# 添加 MySQL YUM 源

官方安装源，这个需要注册账号获取更新，注意是 Oracle 的账号。另附5.7下载的地址，下载之后通过 rpm -Uvh 安装。

$wget 'https://dev.mysql.com/get/mysql57-community-release-el7-11.noarch.rpm'
$sudo rpm -Uvh mysql57-community-release-el7-11.noarch.rpm
$yum repolist all | grep mysql
mysql-connectors-community/x86_64 MySQL Connectors Community                  36
mysql-tools-community/x86_64      MySQL Tools Community                       47
mysql57-community/x86_64          MySQL 5.7 Community Server                 187

官方镜像下载的速度太慢，可以换成国内镜像：

http://mirrors.ustc.edu.cn/mysql-ftp/Downloads/

http://uni.mirrors.163.com/mysql/Downloads/

找到对应的.rpm 源链接，下载后安装。

# 可以将这些包下载后复制到 /var/cache/yum/x86_64/7/mysql57-community/packages/目录，或者到目录下面执行下载。
# 然后再次执行 `yum install mysql-community-server`安装

# 选择安装版本

如果想安装最新版本的，直接使用 yum 命令即可

$sudo yum install mysql-community-server

如果想要安装 5.6 版本的，有2个方法。命令行支持 yum-config-manager 命令的话，可以使用如下命令：

$ sudo dnf config-manager --disable mysql57-community
$ sudo dnf config-manager --enable mysql56-community
$ yum repolist | grep mysql
mysql-connectors-community/x86_64 MySQL Connectors Community                  36
mysql-tools-community/x86_64      MySQL Tools Community                       47
mysql56-community/x86_64          MySQL 5.6 Community Server                 327

或者直接修改 /etc/yum.repos.d/mysql-community.repo 这个文件

# Enable to use MySQL 5.6
[mysql56-community]
name=MySQL 5.6 Community Server
baseurl=http://repo.mysql.com/yum/mysql-5.6-community/el/7/$basearch/
enabled=1 #表示当前版本是安装
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

[mysql57-community]
name=MySQL 5.7 Community Server
baseurl=http://repo.mysql.com/yum/mysql-5.7-community/el/7/$basearch/
enabled=0 #默认这个是 1
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

通过设置 enabled 来决定安装哪个版本。

设置好之后使用 yum 安装即可。

如报如下的错误

ERROR 2002 (HY000): Can't connect to local MySQL server through socket '/tmp/mysql.sock' (2)
# 修改mysql配置文件将套接字指定为mysql默认分配的就行了,
# 修改`/etc/my.cnf`,将`socket`的值改为`/var/run/mysqld/mysqld.sock`即可.

# 修改root密码

成功登陆执行下面Sql语句：

USE mysql;  --选择数据库
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的新密码' --更改密码
SELECT User, Host, plugin FROM mysql.user; --选择显示用户表
UPDATE user SET plugin='mysql_native_password' WHERE User='root';
FLUSH PRIVILEGES; --刷新权限列表
exit;

重启服务systemctl restart mysql 然后使用新的密码登陆试试mysql -u root -p

# centos查看和修改密码

CetnOS 8 Stream版本的系统中安装的mysql默认用户root的密码为空，可直接登陆。

# 通过日志查看

grep 'temporary password' /var/log/mysqld.log

# 修改密码

mysql -u root -p登陆

这里因为mysql的新版本限制了密码复杂度，所以我们需要设置一个稍微复杂的密码

set password=password('这里输入你想改的密码');  --修改当前用户
--或者使用丰面的命令指定修改密码
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的新密码'

如果修改密码时提示下面的错误，表示密码不会使密码强度安全等级，需要调整密码安全设置或者符合复杂强度的密码

ERROR 1819 (HY000): Your password does not satisfy the current policy requirements

# 密码强度等级

set global validate_password_policy=0;  -- 0-2共3个强度等级，0为最低；
set global validate_password_mixed_case_count=3; --至少要包含的小写字母个数和大写字母个数。
set global validate_password_number_count=3;  --包含的数字个数；
set global validate_password_special_char_count=0;  --特殊字符数。
set global validate_password_length=3; ---支持的最短长度

# 开启远程连接

use mysql;
update user set Host = '%' where Host = 'localhost' and User='root'; 
flush privileges; ---刷新
